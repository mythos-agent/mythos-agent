# Getting Support

Thanks for using sphinx-agent! Here's how to get help.

## Quick Guide

| I want to... | Best channel |
|---|---|
| Ask a usage question | [GitHub Discussions → Q&A](https://github.com/sphinx-agent/sphinx-agent/discussions/categories/q-a) |
| Share an idea or feature request | [GitHub Discussions → Ideas](https://github.com/sphinx-agent/sphinx-agent/discussions/categories/ideas) |
| Report a reproducible bug | [Open a bug report issue](https://github.com/sphinx-agent/sphinx-agent/issues/new?template=bug_report.yml) |
| Request a new feature | [Open a feature request issue](https://github.com/sphinx-agent/sphinx-agent/issues/new?template=feature_request.yml) |
| Report a security vulnerability | Email **security@sphinx-agent.dev** — see [SECURITY.md](./SECURITY.md) |
| Report a Code of Conduct violation | Email **conduct@sphinx-agent.dev** |

## Before Opening an Issue

1. Search existing [issues](https://github.com/sphinx-agent/sphinx-agent/issues) and [discussions](https://github.com/sphinx-agent/sphinx-agent/discussions) — your question may already be answered.
2. Check the [README](./README.md) and [CONTRIBUTING.md](./CONTRIBUTING.md).
3. Try the latest version: `npx sphinx-agent@latest scan`.

## Response Expectations

sphinx-agent is maintained by volunteers. We aim to:

- Acknowledge bug reports within **3 business days**
- Acknowledge security reports within **48 hours** (see [SECURITY.md](./SECURITY.md))
- Review pull requests within **7 business days**

Response times may be slower during holidays or heavy-load weeks. Politely pinging a stale thread after two weeks is fine.

## Commercial Support

There is currently no paid support tier. If your organization relies on sphinx-agent and needs guaranteed response times or custom scanner development, open a discussion — we'd like to hear from you.
